# LLVM IR Syntax & Control Flow Guide

To really understand LLVM, you must be able to read the `.ll` files. Here is the breakdown of the syntax and the most critical concept: Control Flow in SSA.

## 1. Type System
LLVM is strongly typed. You cannot treat an integer as a pointer without an explicit cast.
- **Integers:** `i1` (bool), `i8` (byte), `i32` (int), `i64` (long), `i128`, etc.
- **Floating Point:** `float` (32-bit), `double` (64-bit).
- **Pointers:** `ptr` (Opaque pointer in newer LLVM, `i32*` in older versions).
- **Arrays:** `[4 x i32]` (an array of 4 integers).
- **Vectors:** `<4 x float>` (SIMD vector of 4 floats).

## 2. Memory Instructions
Registers (like `%1`, `%result`) are immutable (SSA). To change values (like variables in C), you use memory.
- `alloca`: Allocates memory on the stack (local variable). Returns a pointer.
- `store`: Writes a value to a memory address.
- `load`: Reads a value from a memory address.

**Example (C -> IR):**
```c
// C
int x = 5;
x = 6;
```
```llvm
; LLVM IR
%ptr = alloca i32        ; Allocate space for x
store i32 5, ptr %ptr    ; Write 5
store i32 6, ptr %ptr    ; Overwrite with 6 (this is allowed in memory!)
%val = load i32, ptr %ptr ; Load the value (6) into a register
```

## 3. Control Flow & The PHI Node
This is the trickiest part of LLVM.
Since you cannot change a register (e.g., you can't say `x = x + 1`), how do loops work?

**The Logic:**
- You break code into **Basic Blocks** (Labelled sections).
- You jump between blocks using `br` (branch).
- When two different paths merge (like after an `if/else` or at the top of a `loop`), you need to know *which* value to use.

**The `phi` instruction:**
"If we came from Block A, use value X. If we came from Block B, use value Y."

**Example: The Loop Counter**
We want: `for (int i = 0; i < 10; i++)`

```llvm
entry:
  br label %loop

loop:
  ; The Magic PHI Node
  ; %i = 0 if coming from 'entry'
  ; %i = %next_i if coming from 'loop' (the backedge)
  %i = phi i32 [ 0, %entry ], [ %next_i, %loop ]

  ; Do work...

  ; Increment
  %next_i = add i32 %i, 1

  ; Compare
  %cond = icmp slt i32 %next_i, 10
  br i1 %cond, label %loop, label %exit

exit:
  ret void
```

## 4. Key Instructions Dictionary
- `add`, `sub`, `mul`, `sdiv` (signed div): Arithmetic.
- `icmp eq`, `icmp slt` (signed less than): Integer comparisons.
- `fcmp oeq`, `fcmp olt`: Float comparisons.
- `br label %dest`: Unconditional jump.
- `br i1 %cond, label %true, label %false`: Conditional jump.
- `call`: Call a function.
- `ret`: Return from function.
